# CLAUDE.md

## Project

- **Name:** open-game-system
- **Description:** Monorepo for the Open Game System — a platform enabling web games to leverage native mobile capabilities (push notifications, cloud rendering/streaming) through the OGS companion app.
- **Tech stack:** TypeScript, pnpm workspaces, Turbo, Vitest (packages), Jest (mobile app), Hono + Cloudflare Workers (API), Expo/React Native (mobile), tsup (package builds)
- **Build:** pnpm 9.15+, Node 20+, Turbo

## Core Principles

- **pnpm workspaces** with `workspace:*` for internal deps — no publishing needed for internal consumption
- **Independent builds**: Each package builds independently via tsup (ESM + CJS + DTS)
- **Services deploy to Cloudflare Workers**: The `services/api` package deploys via wrangler
- **Apps build via EAS Build**: Managed Expo workflow for `apps/mobile`
- **TDD**: Red-green-refactor, always. Write the test first, see it fail, implement, see it pass.

## Key Conventions

- **Package naming:** `@open-game-system/<name>`
- **Internal deps:** `workspace:*` protocol in `package.json`
- **Test runner:** Vitest for packages, Jest for mobile app
- **Build orchestration:** Turbo with dependency-aware task graph
- **No publishing yet** — defer until API is stable

## Workspace Layout

```
apps/       — deployable applications (mobile)
packages/   — shared libraries (app-bridge-*, notification-kit-*, stream-kit-*)
services/   — backend services (api)
examples/   — demo/reference apps
docs/       — cross-project documentation
```
