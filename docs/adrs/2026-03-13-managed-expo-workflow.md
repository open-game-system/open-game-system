# Managed Expo Workflow for Mobile App

**Date:** 2026-03-13
**Status:** Accepted

## Context

Expo apps can use managed workflow (ios/android directories gitignored, regenerated by `expo prebuild`) or bare workflow (ios/android committed, manually maintained).

## Decision

Use managed workflow. The `ios/` and `android/` directories are gitignored and regenerated by `expo prebuild` in CI (EAS Build).

## Consequences

- No merge conflicts on generated native code
- Cleaner diffs — only source changes appear in PRs
- EAS Build runs `expo prebuild` automatically
- Cannot write custom Swift/Kotlin native modules (must use Expo SDK + config plugins)
- Must re-prebuild after dependency changes that affect native code
